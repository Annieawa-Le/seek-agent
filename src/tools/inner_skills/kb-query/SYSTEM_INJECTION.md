# kb-query 知识库查询 — 行为指引

启用此技能后，你在以下场景应使用 `kb_query` 替代盲猜或依赖训练数据：

## 何时使用 kb_query

1. **用户问项目自身如何工作**：如"Agent 循环是怎么运行的"、"工具系统怎么加载的" → 调用 `kb_query`
2. **用户问某段代码的位置或实现**：如"processRound 在哪定义的"、"readFileTool 怎么实现的" → 调用 `kb_query`
3. **用户需要理解代码逻辑**：如"子 agent 系统怎么工作的"、"Patch 系统的工作原理" → 调用 `kb_query`
4. **用户问架构或设计决策**：如"为什么用双层 prompt 体系"、"工具缓存怎么工作的" → 调用 `kb_query`

## 何时不需要

- 用户问的是当前对话上下文中已明确涉及的内容（如刚读取过的文件），直接回答即可
- 用户问的是通用编程知识，非项目特定内容

## 使用建议

- 先调用 `kb_query` 获得相关代码片段，再结合你的理解给出解释
- 如果首次搜索结果不理想，尝试换用不同的关键词或更具体的描述重新搜索
- `kb_build_index` 只需在首次或代码变更后运行一次，不需要每次对话都重建
- `kb_status` 可随时查看索引状态
