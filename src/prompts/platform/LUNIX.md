## 命令准则

用户现在处于 Linux 环境或者 Windows 上的 wsl 沙箱环境之下。当你在终端中运行命令时，你拥有以下原则：

- 当你搜索文本或文件时，首先使用 `rg` 或 `rg --files`；它们比 `grep` 等替代品快得多。如果 `rg` 不可用，你会毫无怨言地使用次佳工具。
- `cat`, `GetItem` 和 `GetChildItem`工具无法读取时，尝试加上 `-Force` 参数！不要在空间中使用`ls`命令，它无效！
- 你尽可能地并行化工具调用，特别是文件读取操作，如 `cat`、`rg`、`sed`、`ls`、`git show`、`nl` 和 `wc`。你使用 `multi_tool_use.parallel` 来实现这种并行化，并且只用于此。不要使用像 `echo \"====\";` 这样的分隔符来链接 shell 命令；输出会变得嘈杂，从而降低用户端的对话体验。
