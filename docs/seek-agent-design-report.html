01 Token 架构

三层 Token 设计系统

遵循 UI/UX Pro Max 规范的三层 Token 架构：原始值 → 语义别名 → 组件专用

Primitive
--gold-500: #C4A246
↓
Semantic
--accent: var(--gold-500)
↓
Component
--btn-bg: var(--accent)

核心原则

  • 绝不使用原始 hex —— 始终引用 Token
  • 语义层支持主题切换（亮/暗）
  • 组件 Token 支持逐组件定制
  • 使用 HSL 格式便于透明控制
  • 每个 Token 记录用途文档
/* Component Token 示例 */ --send-btn-bg: var(--accent); --send-btn-hover: var(--accent-hover); --msg-user-bg: rgba(212,168,67,0.06); --msg-agent-bg: rgba(255,255,255,0.03);

03 排版与字体

Inter + JetBrains Mono 双字体系统

清晰的技术调性，契合开发者工具的产品气质。动态缩放比例基于 clamp() 实现响应式。

字体家族

Inter
正文字体 · 300-700 字重
JetBrains Mono
等宽字体 · 代码、数据、终端

字号层级

/* 动态缩放 — 基于 clamp */ --text-xs: 0.75rem; /* 标签/辅助 */ --text-sm: 0.875rem; /* 正文小 */ --text-base: 1rem; /* 正文 */ --text-lg: 1.125rem; --text-xl: 1.25rem; --h3: clamp(1.1rem, 2vw, 1.4rem); --h2: clamp(1.4rem, 3vw, 1.8rem); --h1: clamp(1.8rem, 4vw, 2.8rem);

行高与字重规范

  • 正文行高 1.6–1.75，代码行高 1.5
  • 标题字重 600–700，正文 400，标签 500
  • 代码内联背景 rgba(168,85,247,0.12)
  • 代码块背景 rgba(0,0,0,0.3)
  • 标题色使用 金色 突出
  • 移动端最小 16px 正文（防 iOS 缩放）

05 组件设计规范

组件 状态与变体

每个组件定义默认、悬停、激活、禁用四种状态，确保交互反馈一致。

阴影 none --glow-gold none none

过渡动画

  • 默认过渡：150–200ms cubic-bezier(0.16, 1, 0.3, 1)
  • 消息入场：250ms fade-in + translateY(6px)
  • 光标闪烁：1s step-end 无限循环
  • 状态指示点脉冲：1.5s ease-in-out 无限
  • 尊重 prefers-reduced-motion
06 深色模式实现

Dark Mode (OLED) 独家深色

纯深色模式设计，无亮色回退。针对 OLED 屏幕优化，低蓝光护眼，高对比度确保可读性。

背景色分层策略

--bg-deepest
侧边栏 · 深层背景 · 接近纯黑
--bg-base
主体背景 · 主工作区
--bg-elevated
抬高层 · 消息区背景
--bg-surface
卡片表面 · 面板

金色光晕系统

/* Form 渐变光晕 */ background: linear-gradient(90deg, transparent, rgba(212, 168, 67, 0.25), transparent); /* Header 底部渐变线 */ background: linear-gradient(135deg, var(--slate-50), var(--gold-400));

对比度验证

主文字 ← 背景 ✓ 15.7:1
次文字 ← 背景 ✓ 10.4:1
弱文字 ← 背景 ✓ 6.8:1

毛玻璃效果层

  • Header：backdrop-filter: blur(12px)
  • 背景透光：rgba(21,18,16,0.88)
  • 输入栏：rgba(13,10,8,0.5) + blur(8px)
  • 底部导航：rgba(13,10,8,0.85) + blur(12px)

关键设计决策

"暖调深色 + 金色点缀"
比纯蓝黑更温暖，比纯灰黑更有质感。金色作为开发工具的品牌色，传递"价值感"与"精准"。
07 UI 风格展示

风格定调 · Dark Developer Tool

电影级深色 · 开发者工具 · 微玻璃质感 · AI 原生体验。源自项目现有 style.css 的设计语言。

✦ 设计风格关键词

Dark Mode OLED 电影级深色 微玻璃质感 AI 原生 金色点缀 暖调深棕 高对比度 代码友好 终端美学 低蓝光

✦ 反模式清单（避免）

  • 不使用 emoji 作为图标（使用 SVG）
  • 纯色深色主题，无亮色默认
  • 避免低对比度（< 4.5:1）
  • 所有可点击元素有 cursor: pointer
  • 不依赖单一颜色传达信息
  • 缓慢/卡顿性能

✦ 数据看板风格统计

✦ 交互反馈模式

  • 按钮悬停：上移 1px + 光晕 + 亮色
  • 卡片悬停：上移 4px + 边界高亮 + 顶部渐变色条
  • 输入聚焦：金色边框 + 外发光
  • 消息入场：淡入 + 上移（250ms）
  • 禁用态：灰色 + 无交互
08 可访问性与预交付检查

CRITICAL · 可访问性优先

遵循 UI/UX Pro Max 的优先级规则，可访问性为最高优先级（Critical）。

可访问性检查清单

  • 正常文本最小对比度 4.5:1，大文本 3:1
  • 交互元素必须有可见焦点环
  • 图标按钮必须有 aria-label
  • 不仅靠颜色传达信息（加图标/文字）
  • 支持 prefers-reduced-motion
  • 触摸目标最小 44x44pt
  • 触摸目标间至少 8px 间距

预交付检查清单

✓无 emoji 用作图标（使用 SVG）
✓语义色值统一使用 Token
✓所有可触摸元素有按压反馈
✓正文字体对比度 ≥4.5:1
✓深色模式独立验证
✓已在小屏、大屏验证响应式
⏱
动画规范
时长 150-300ms · cubic-bezier 缓出 · 尊重运动减少偏好
